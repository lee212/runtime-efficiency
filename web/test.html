Draft page for cloud run-time efficiency measurement

As a first step of measuring the preparation stages for cloud environments, we have collected timestamp between the processes of RAIN and the processes are:

  • Eucalyptus
    1. Image preparation - time between available image and Rain client launched.
    2. Launching images - time taken for launching VM instances and waiting them to be running.
    3. SSH accessibility - time for allocating public IP/private IP addess(es). The IP address(es) should be accessible via SSH and this measured time-value indicates between requests and availability of the address(es).
    4. Total walltime - entire processing time
  • OpenStack
    1. IP address association - time for associatin VM instance(s) and ip address(es)
    2. Mounting/configuring directories - Configuring ssh in VM and mounting home directory (using sshfs) in /tmp
    3. Running jobs - ? (/tmp/N/u/jdiaz/exit.sh)
    4. Rest of them are same as Eucalyptus

  • Spiderweb chart
    • * Logarithm is applied to display time-value.
    * Legend shows like this 'client' + '_' + 'platform' + '-' instance + 'number of vm requested' + '_' + 'sequence number of request'
    e.g. rain_euca-1_1 - first request(try) of Rain client for Eucalyptus with 1 VM instance.
    * In case the graph doesn't show up, please refresh(F5) the browser. Highchart sometimes doesn't load html element correctly.
